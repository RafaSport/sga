# Sistema de Gestão de Abrigos

Rede de Acolhimento — Prefeitura do Recife

## Etapa 1 — Visão Geral do Projeto

### 1. Identificação do Projeto

- **Nome do Projeto:** SGA — Sistema de Gestão de Abrigos
- **Organização:** Prefeitura do Recife — Secretaria de Assistência Social
- **Abrangência:** Rede de Abrigos do Recife
- **Público Atendido:** Pessoas em situação de acolhimento institucional
- **Versão do Documento:** 1.0 — Rascunho para revisão
- **Data:** Abril de 2026

### 2. Contexto e Problema

- Registro em papel e fichas físicas
- Dificuldade de acesso rápido ao histórico
- Risco de perda de documentos
- Impossibilidade de compartilhamento em tempo real
- Retrabalho em relatórios
- Falta de visão consolidada da rede
- Dificuldade no controle de medicações e agendamentos

### 3. Objetivos

- **Geral:** Sistema web para gestão integrada da rede
- **Específicos:** prontuário digital, relatórios técnicos, controle de medicações, gestão de documentos, movimentações, painel gerencial, controle de acesso por perfil

### 4. Usuários do Sistema

| Perfil              | Função Principal  | Nível de Acesso      |
| ------------------- | ----------------- | -------------------- |
| Gerente             | Visão geral       | Completo             |
| Assistente Social   | Relatórios        | Técnico              |
| Psicólogo(a)        | Atendimentos      | Técnico              |
| Administrativo      | Medicações        | Administrativo       |
| Educador Social     | Cotidiano         | Operacional          |
| Cuidador            | Intercorrências   | Operacional limitado |
| Coordenação da Rede | Visão consolidada | Completo rede        |

### 5. Escopo

- **Dentro:** cadastro, prontuário, documentos, medicações, agendamentos, painel gerencial, controle de acesso, sistema web responsivo
- **Fora:** integração com sistemas externos, app nativo, módulo financeiro, emissão de documentos oficiais

### 6. Premissas e Restrições

- Internet disponível em cada abrigo
- Responsável por manter dados
- LGPD obrigatória
- Desenvolvimento incremental
- Recursos limitados
- Simplicidade para usuários sem formação técnica
- Compatibilidade com computadores simples
